# 最简易部署文档

适用：一台 Windows、一个大 QMT 账户。日常外置侧只运行一个文件：

```powershell
.\start_external_windows.ps1
```

大 QMT 必须已登录，且其中的 Helper 策略必须已启动；启动器不会操作大 QMT 的登录或 GUI。

## 首次部署（只做一次）

### 1. 准备配置

安装 **CPython 3.12 x64**，然后打开 PowerShell：

```powershell
cd "D:\xuanling\全自动\参考\开源项目\一台windows大QMT内置python桥接本机连接外置方案"
.\setup_venv.ps1
Copy-Item .env.example .env
Copy-Item coordinator_config.example.json coordinator_config.json
& .\.venv\Scripts\python.exe -c "import secrets; print(secrets.token_hex(32)); print(secrets.token_urlsafe(24))"
notepad .env
notepad coordinator_config.json
```

在 `.env` 中只填写这几项：

```dotenv
QMT_LOCAL_AUTH_TOKEN=<上面第一行的 64 位令牌>
QMT_LOCAL_ACCOUNT_ENABLED=true
QMT_LOCAL_ACCOUNT_NAME=account_main
QMT_LOCAL_ACCOUNT_ID=<大QMT真实资金账号>
QMT_LOCAL_ACCOUNT_TYPE=STOCK
```

普通账户使用 `STOCK`；信用账户使用 `CREDIT`。首次部署保持以下两项为 `false`：

```dotenv
QMT_LOCAL_HELPER_ENABLE_TRADING=false
QMT_LOCAL_HELPER_ENABLE_CANCEL_ORDER=false
```

在 `coordinator_config.json` 中把 `alpha.auth_token` 改为上面第二行的随机字符串。保持示例中的 `worker.enabled=false`，即可先不自动启动策略。

> `.env` 和 `coordinator_config.json` 含有凭据：不要提交、不要发送给他人。

### 2. 部署大 QMT Helper

先在大 QMT 中停止旧的桥接策略，再执行：

```powershell
.\generate_helper.ps1 -Deploy -ConfirmStoppedStrategy
```

在大 QMT 中，新建/更新 Python 策略并加载：

```text
C:\Quant\QmtLocalBridge\helpers\account_main\bigqmt_loader.py
```

若修改过 `.env` 中的 Helper 安装路径或账户别名，以上路径随之变化。策略绑定的账户必须与 `.env` 完全一致；启动该策略并保持大 QMT 登录。

## 日常启动

确认大 QMT 已登录、`bigqmt_loader.py` 策略正在运行后，在项目根目录执行：

```powershell
.\start_external_windows.ps1 -NoWorkers
```

看到 `External Windows environment is ready.` 即成功。此命令会自动完成外置侧的预检、Gateway 和唯一 Coordinator 启动。

日常查看和停止：

```powershell
.\start_external_windows.ps1 -Status
.\start_external_windows.ps1 -Stop
```

不要再单独运行 `start_gateway.ps1`，也不要让多个策略直接连接 Gateway。

## 要自动启动策略时

在 `coordinator_config.json` 中配置策略程序，且把对应的 `worker.enabled` 改为 `true`；策略只能连接 Coordinator `127.0.0.1:9560`，不能读取根 `.env` 或直接连接 Gateway。然后执行：

```powershell
.\start_external_windows.ps1
```

## 开启真实下单前

先完成模拟账户/只读验证。需要真实下单或撤单时：停止外置环境和大 QMT Helper 策略，在 `.env` 中将相应开关改为 `true`，重新执行 `generate_helper.ps1 -Deploy -ConfirmStoppedStrategy`，再重新加载并启动 Helper。只改 `.env` 不会改变已运行的 Helper。

启动失败时，先执行：

```powershell
.\start_external_windows.ps1 -Status
```

最常见原因只有三个：大 QMT 未登录、Helper 策略未运行，或 `.env` / `coordinator_config.json` 仍有示例值。
