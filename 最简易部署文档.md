# 最简易部署文档

项目目录：

```text
D:\xuanling\全自动\参考\开源项目\一台windows大QMT内置python桥接本机连接外置方案
```

固定启动顺序：大 QMT 登录并启动 Loader 策略 → `start_gateway.ps1` → 唯一外置策略进程。

## 1. 创建项目虚拟环境

确保 Windows 已安装 CPython 3.12 x64。无需查找或记录系统 `python.exe` 路径；打开 PowerShell 后只执行项目脚本：

```powershell
Set-Location -LiteralPath "D:\xuanling\全自动\参考\开源项目\一台windows大QMT内置python桥接本机连接外置方案"
.\setup_venv.ps1
```

脚本会在项目根目录创建或复用 `.venv`，并通过 `.pth` 离线暴露 `外置策略API`。不需要激活虚拟环境，也不要对本项目执行 `pip install -e`。以后所有普通 Python 命令均使用 `& '.\.venv\Scripts\python.exe'`。

## 2. 生成并修改 `.env`

打开 PowerShell：

```powershell
Set-Location -LiteralPath "D:\xuanling\全自动\参考\开源项目\一台windows大QMT内置python桥接本机连接外置方案"
Copy-Item -LiteralPath ".env.example" -Destination ".env"
& '.\.venv\Scripts\python.exe' -c "import secrets; print(secrets.token_hex(32))"
notepad .env
```

把令牌命令输出的 64 位十六进制字符串填入 `.env`。只需修改或确认以下配置：

| 配置 | 要填写的值 |
|---|---|
| `QMT_LOCAL_PYTHON_EXE` | 固定为 `.venv\Scripts\python.exe`，禁止改成系统 Python 绝对路径 |
| `QMT_LOCAL_AUTH_TOKEN` | 刚生成的 64 位令牌，不能保留全零示例 |
| `QMT_LOCAL_ACCOUNT_ENABLED` | `true` |
| `QMT_LOCAL_ACCOUNT_NAME` | ASCII 账户别名；默认 `account_main` 可直接使用 |
| `QMT_LOCAL_ACCOUNT_ID` | 大 QMT 中的真实资金账号 |
| `QMT_LOCAL_ACCOUNT_TYPE` | 普通账户填 `STOCK`，信用账户填 `CREDIT` |

以下配置首次部署保持不变：

```dotenv
QMT_LOCAL_BIND_HOST=127.0.0.1
QMT_LOCAL_TCP_PORT=9550
QMT_LOCAL_HELPER_ENABLE_TRADING=false
QMT_LOCAL_HELPER_ENABLE_CANCEL_ORDER=false
```

默认 `C:\Quant\QmtLocalBridge\...` 路径可写时无需修改。若必须修改，所有路径应使用本地盘符、ASCII 字符，并保持各目录互不嵌套。

## 3. 生成并部署大 QMT 文件

先在大 QMT 中停止旧的桥接策略；首次部署没有旧策略时直接执行：

```powershell
Set-Location -LiteralPath "D:\xuanling\全自动\参考\开源项目\一台windows大QMT内置python桥接本机连接外置方案"
.\generate_helper.ps1 -Deploy -ConfirmStoppedStrategy
```

不要运行或加载 `大QMT内置python\src` 下的模板。需要加载的文件是：

```text
<QMT_LOCAL_HELPER_INSTALL_ROOT>\<QMT_LOCAL_ACCOUNT_NAME>\bigqmt_loader.py
```

使用默认配置时路径为：

```text
C:\Quant\QmtLocalBridge\helpers\account_main\bigqmt_loader.py
```

## 4. 在大 QMT 中启动 Loader

按顺序操作：

1. 登录大 QMT；
2. 新建或更新 Python 策略；
3. 策略文件选择上一步生成的 `bigqmt_loader.py`；
4. 策略绑定的账号 ID 和账户类型必须与 `.env` 一致；
5. 启动策略并保持大 QMT 登录。

## 5. 启动本机 Gateway

新开一个 PowerShell 窗口并保持窗口运行：

```powershell
Set-Location -LiteralPath "D:\xuanling\全自动\参考\开源项目\一台windows大QMT内置python桥接本机连接外置方案"
.\start_gateway.ps1
```

Gateway 必须在大 QMT Loader 策略启动后运行。如果 Helper 尚未 ready，脚本会拒绝启动。

## 6. 最小在线验证

再开一个 PowerShell 窗口：

```powershell
Set-Location -LiteralPath "D:\xuanling\全自动\参考\开源项目\一台windows大QMT内置python桥接本机连接外置方案"
& '.\.venv\Scripts\python.exe' .\示例\query_account.py --query-type ACCOUNT_STATUS
```

返回 JSON 中出现 `"success": true` 且没有 `ready: false` 或 `degraded` 即部署成功。该示例结束后再启动正式策略。

## 7. 启动外置策略

`setup_venv.ps1` 已通过 `.pth` 暴露本项目 API，无需安装。使用项目 `.venv` 启动自己的外置策略；外置策略必须在同一个进程、同一个 `LocalQmtApi` 连接中完成回调处理和下单。一个账户同时只运行一个可靠事件消费者。

若策略另有第三方依赖，只使用 `& '.\.venv\Scripts\python.exe' -m pip install <依赖名>` 安装到项目虚拟环境。

仅验证回调时可以运行：

```powershell
& '.\.venv\Scripts\python.exe' .\示例\callback_client.py --query-on-start ACCOUNT_STATUS
```

正式外置策略启动前先停止这个回调示例，不能让它与正式策略同时连接。

## 8. 开启真实下单或撤单

首次只读联调成功后，按顺序执行：

1. 停止外置策略；
2. 按 `Ctrl+C` 停止 Gateway；
3. 在大 QMT 中停止 Loader 策略；
4. 修改根 `.env`：下单设置 `QMT_LOCAL_HELPER_ENABLE_TRADING=true`，需要撤单再设置 `QMT_LOCAL_HELPER_ENABLE_CANCEL_ORDER=true`；
5. 再次运行 `.\generate_helper.ps1 -Deploy -ConfirmStoppedStrategy`；
6. 按第 4、5、7 节顺序重新启动。

只修改 `.env` 不会改变已经运行的 Helper，必须重新生成、部署并重载 Loader。

## 9. 以后每次启动的文件顺序

```text
1. 大 QMT 客户端登录
2. <QMT_LOCAL_HELPER_INSTALL_ROOT>\<QMT_LOCAL_ACCOUNT_NAME>\bigqmt_loader.py
3. 项目根目录\start_gateway.ps1
4. 使用项目 `.venv` 启动唯一外置策略 Python 文件
```
